    Overview

    Generating API Key

    You will need to create an API key on the BTSE platform before you can use authenticated APIs. To create API keys, you can follow the steps below:

    • Login with your username / email and password into the BTSE website
    • Click on “Account” on the top right hand corner
    • Select the API tab
    • Click on “New API” button to create an API key and passphrase. (Note: the passphrase will only appear once)
    • Use your API key and passphrase to construct a signature.

    Authentication

    • API Key (btse-api)

      • Parameter Name: btse-api, in: header. API key is obtained from BTSE platform as a string

    • API Key (btse-nonce)

      • Parameter Name: btse-nonce, in: header. Representation of current timestamp in long format

    • API Key (btse-sign)

      • Parameter Name: btse-sign, in: header. A composite signature produced based on the following algorithm: Signature=HMAC.Sha384 (secretkey, (urlpath + btse-nonce + bodyStr)) (note: bodyStr = '' when no data):

    POST /api/v1/address

    Creates a new wallet address

    Request Parameters

    Name Type Required Description
    username string Yes Subaccount identifier
    coin string Yes Coin protocol. Valid values are:
    USDT-ERC20
    USDT-TRC20
    referenceid string Yes Client reference ID
    isExpire boolean No Indication if address is to be expired. If value is set to false, then only a maximum of 100 addresses can be created. Default is true
    callbackUrl string Yes Callback URL when a deposit is received on this address
    data string Yes Custom data. Can send any payload that you want to be sent over in the callback.

    Response Content

    Name Type Required Description
    address string Yes Deposit address

    GET /api/v1/transactions

    Retrieves transaction records

    Request Parameters

    Name Type Required Description
    username string Yes Subaccount username
    from string Yes Date format in 'yyyy-MM-dd HH:MI:SS'
    to string Yes Date format in 'yyyy-MM-dd HH:MI:SS'
    coin string Yes Coin currency

    Response Content

    Name Type Required Description
    username string Yes Subaccount username
    txnDate string Yes Transaction date
    coin string Yes Coin currency
    txnRef string Yes BTSE internal transaction reference
    transactionType integer Yes Transaction type with values:
    1: Deposit
    2: Withdraw
    5: Transfer In
    6: Transfer Out
    amount double Yes Transaction amount
    fees double Yes Transaction fees
    txid string Yes Blockchain transaction ID
    address string Yes Blockchain address
    status integer Yes Transaction status with values:
    1: Open
    2: Locked
    10: Completed
    14: Expired
    15: Failed
    confirmations string Yes Blockchain confirmation list

    POST /api/v1/withdraw

    Perform withdrawals

    Request Parameters

    Name Type Required Description
    coin string Yes Currency coin
    amount double Yes Withdrawal amount
    username string Yes Subaccount username
    address string Yes Withdrawal address
    referenceId string Yes Client reference ID
    callbackURL string Yes Callback URL
    data string Yes Payload data from client, this will be a passback
